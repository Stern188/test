# 使用Bootstrap ## 1 在ngCloud中使用 bootstrap-material-design 为了保持系统整体样式统一性，ngCloud中默认使用的是[material design](http://fezvrasta.github.io/bootstrap-material-design)样式的bootstrap。 实际上使用bootstrap主要原因有两个： - angular的material2组件不完整 - bootstrap布局效果更好
如后后续angular官方material2和flexlayout更加完善，ngCloud会去掉对bootstrap的支持。所以建议开发人员优先使用material2与flexlayout。
如果要使用bootstrap只需要直接使用bootstrap的样式即可。不建议使用bootstrap对全页面进行布局控制,特别是响应式布局，以免引起与flexlayout的冲突。 如以下代码： ``` Default Primary Success Info Warning Danger ``` 输出效果： Default Primary Success Info Warning Danger ## 2 依赖的库 目前依赖的库包括(已在package.json中，默认已安装)： - bootstrap - bootstrap-sass - bootstrap-material-design 对于.angular-cli.json的修改包括： ``` "styles": [ "../node_modules/bootstrap-material-design/dist/css/ripples.min.css" ], "stylePreprocessorOptions": {{'{'}} "includePaths": [ "../node_modules/bootstrap-sass/assets/stylesheets" ] {{'}'}}, "scripts": [ "../node_modules/jquery/dist/jquery.min.js", "../node_modules/arrive/minified/arrive.min.js", "../node_modules/bootstrap-material-design/dist/js/material.js", "../node_modules/bootstrap-material-design/dist/js/ripples.min.js" ] ``` 样式可以直接在`.angular-cli.json`的styles中引入bootstrap与bootstrap-material-design的css,也可以在ngCloud的`styles.scss`中`@import`进来。 目前系统采用import方式。通过import可以自定义样式颜色。直接加载css要灵活一些。 ## 3 不使用bootstrap 如果不使用bootstrap 可以删除bootstrap npm包节省空间，避免css污染: ``` npm uninstall bootstrap -S npm uninstall bootstrap-sass -S npm uninstall bootstrap-material-design -S ``` 删除`style.scss`中的引用 ``` @import 'sass/bootstrap-mat'; //bootstrap-material-design ``` 删除`.angular-cli.json`中对于bootstrap引入的所有配置信息
